# Evaluation Summary Report v2

## 1. Diagnosis Findings
Previous evaluation run produced M2 Visual Judge results with zero variance: all 10 videos across all 3 dimensions evaluated to exactly 4.0 ± 0.0.
Phase 1 diagnosis confirmed Hypothesis A (model bias toward '4'). Specifically:
- Raw model response outputs successfully returned syntactically valid JSON structures, bypassing fallbacks.
- Rationales generated by the model contained video-specific details (indicating genuine semantic processing, not simple templates), but the scores themselves were biased to 4.

## 2. Fix Applied
We applied the fix under Branch 2.A:
- **Prompt Revision**: Revised system prompt `JUDGE_VISUAL_SYSTEM` with strict calibration anchors on the 1-5 scale (1 = Severe failures, 2 = Notable issues, 3 = Acceptable, 4 = Good, 5 = Excellent). Replaced instructions to prompt the model to use the full range.
- **Rationale Formatting**: Added instructions to separate observation rationales for each dimension using `' | '`.
- **Sampling Parameters**: Set `do_sample=True`, `temperature=0.7`, `top_p=0.9`, and `repetition_penalty=1.05` to break greedy decoding patterns.
- **Multi-Sampling**: Configured the pipeline to query the visual judge 3 times per video to capture model variance.
