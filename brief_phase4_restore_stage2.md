## Stage 2.2: Adapt assembler untuk consume per-group format

### Current state assembler

`src/phase5_assemble.py` di v8 reads sentence-level matches (`scene_matches_{method}.json`) dan cut video per-sentence.

### Target behavior

Adapt untuk:
1. Read `p4_assignments.json` (list of `Assignment` objects)
2. Process per group:
   - **For `action="retrieve"` groups:** Extract video segment dari source video berdasarkan group's scene assignment. Use group's locked scene + best frame info.
   - **For `action="generate"` groups:** **Leave placeholder for now** — Phase 5 LTX module akan provide generated clip. Assembler should:
     - Check if generated clip exists at expected path (e.g. `data/intermediate/{video_id}/generated/group_{group_id}.mp4`)
     - If exists: use it
     - If not exists: **fallback to retrieval** (use group's best-matching scene from source video) and log warning
3. Concatenate clips in order with audio per group
4. Optional subtitle overlay (already supported, preserve)

### Implementation notes

- **Don't implement Phase 5 LTX generation yet.** That's separate task. This brief only prepares assembler to consume LTX output later.
- Preserve all v8 audio/subtitle handling logic.
- Audio per group: use TTS output already generated by Phase 3 (concat per-sentence wavs within a group).
- Output structure: `output_dir / {video_id} / final_summary.mp4`
