# Routing Rules for Animation Policy (Content-Aware)

Based on the `animation_plan.json` generated by the pipeline's classification logic:

| Shot Characteristic | Animation Method |
| :--- | :--- |
| Contains motion-implying keywords (e.g., water, lava, magma, wind, flow, melt, erupt, wave) | **Wan I2V** (Full generative video) |
| Lacks motion keywords (static, diagrammatic, or landscape shots) | **Ken Burns** (Zoom/pan on static image) |

**What the code actually checks:**
The pipeline uses a predefined list of `dynamic_keywords` to analyze the shot's content (such as its description or key entities). If any of the target dynamic keywords are matched, the shot is classified as dynamic and routed to Wan I2V. If no match is found, it defaults to the Ken Burns effect to preserve structural clarity and save compute.
